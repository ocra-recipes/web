36 ### solvers
37 
38 Ultimately our goal is to implement the most recent convex solvers so we can mix and match control problem formulations with different solver algorithms. Right now we have only implemented a slightly modified version of QuadProg.
39 
40 #### quadprog
41 
42 This library is a QP (Quadratic Program) based on the QuadProg++ project (http://quadprog.sourceforge.net/) which has been slightly modified. In this version, vector and matrix classes are replaced by Eigen classes, in order to use the same definitions as the ocra libraries.

46 #### qpoases
47 
48 Exploits the open-source C++ library qpOASES, which is an implementation of the recently proposed online active set strategy, which was inspired by important observations from the field of parametric quadratic programming (QP). It has several theoretical features that make it particularly suited for model predictive control (MPC) applications but also as a QP solver. [(ref)](https://projects.coin-or.org/qpOASES)

77 ## Installation
78 
79 **WARNING**
80 *This is an experimental set of libs and there are no guarantees that they will not do damage to your computer. We take no responsibility for what happens if you install them. That being said, if you follow these instructions you should be fine.*
81 
82 Okay that's out of the way... phew!
83 
84 **Install to /usr/local**
85 ```
86 git clone https://github.com/ocra-recipes/ocra-recipes.git
87 cd ocra-recipes
88 mkdir build
89 cd build
90 cmake ..
91 sudo make install
92 ```
93 
94 **Install to custom location (example: /home/user/Install)**
95 ```
96 git clone https://github.com/ocra-recipes/ocra-recipes.git
97 cd ocra-recipes
98 mkdir build
99 cd build
100 cmake -DCMAKE_INSTALL_PREFIX=home/user/Install ..
101 make install
102 ```
103 *Note:* If you do the install this way then you must update your environment variables in your `.bashrc` file.
104 
105 **In `.bashrc`**
106 ```shell
107 # ocra-recipes install
108 export OCRA_INSTALL=/home/user/Install
109 export LD_LIBRARY_PATH=${LD_LIBRARY_PATH}:${OCRA_INSTALL}/lib
110 export LIBRARY_PATH=${LIBRARY_PATH}:${OCRA_INSTALL}/lib
111 export PATH=${PATH}:${OCRA_INSTALL}/bin
112 ```

130 ### Notes about OS X
131 :warning: As a recent feature we added rpath support, therefore, by default the flag `OCRA_ICUB_ENABLE_RPATH` is `OFF`. Change it by configuring `ocra-wbi-plugins` as: `cmake -DOCRA_ICUB_ENABLE_PATH=ON ./`. Once we're sure this ''always'' works it will be `ON` by default. Therefore, if you encounter error messages such as:
132 
133 ```bash
134 dyld library not loaded @rpath/libYarpMath.dylib
135 ```
136 
137 Most likely this variable is still `OFF`.
138 
139 :warning: When using XCode to debug your code, make sure you change your LLVM C++ Language settings manually to C++11, by heading to the `Build Settings` of your project, searching for `C++ Language Dialect` and changing it to `C++11 [-std=c++11]`

146 ### Generating the documentation
147 
148 To build the documentation for `orca-recipes` you will need [`doxygen`](http://www.stack.nl/~dimitri/doxygen/index.html). To install `doxygen` on linux run `sudo apt-get install doxygen`.
149 
150 In your `build/` directory (if you have already run `cmake ..` simply run:
151 ```
152 make doc
153 ```
154 **
155 If you run this right now you must ignore the latex errors due to the missing `.sty` files by just holding down `enter` until they are past.
156 **
157 
158 HTML and LaTeX files will be generated in the `build/docs/` directory. Open the file, `index.html`.
